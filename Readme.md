**1. 地圖不可解的清楚特性**

地圖不可解有一些明顯的特性；因為消除的行動主要為單次移除兩格(或四格)，每種顏色的格子數基本上該有偶數個。若有幾種顏色具有奇數個格子，那也需要是偶數種顏色的格子數為奇數 (例如紅色與黃色都有7個格子，其餘顏色都有偶數個)。但是這很依賴統計各個顏色的方塊數量，因此只有搜尋演算法可以快速實踐。(畢竟哪有人在實際玩遊戲時有那個時間數格子數量?)

第二項是地圖一開始與行動過後的可解格子數統計；我們可以將空白格子也賦予一個代表值 (若紅橙黃依序為1,2,3，那空白可以視為0)，每次行動都要經過統計；可以解的格子配對有多少? 正常來講在正式遊戲中因為地圖有解，我們只需要討論不同種消除方式是否會影響到最終結果。但是我們需要分辨 "可以解" 與 "不可解" 的地圖的話，這種指標也需要進行考慮。但是這種方式只能透過實際運算後透過變數數值紀錄來確認，所以僅能作為最一開始地圖狀態的判斷方式；畢竟一個格子都按不了的地圖根本無法玩 !

第三項是連續與包圍；若有一種顏色的格子連續但沒有其他外部格子可供直角或間隔相消，那就明顯無解 (例如紅色有四格，但四格緊密相鄰排列；例如1x4一直線連續排列或者形成2x2正方形)。此外，在邊界的包圍網也需要討論。所謂包圍網就是一個正方形格子四邊都被其他顏色之格子包圍，這樣這個格子是否能被消除的結果就依賴外部其他格子是否能被消除，以此延展。基於偶數消除的規定，在邊界區域以外的包圍網都很有可能被消除 (你很容易找到一個路徑存在的其他格子可以讓你消除；即便它很遠，因為有直角交界消除方法的存在，任意格子都很容易消除) 。但是若將奇數顏色方塊納入考量，情況就有所不同。

# 遇到的困難

由於目前我對於搜尋演算法/最佳化演算法的路徑的認知不夠深入，我需要時間去認識與了解他們如何運作，以及怎麼在程式碼中實踐(目前沒有任何想法...)。目前我在考慮以下路徑:

1. DFS Backtracking (50格地圖的基礎推導)
這個就是在做資料樣本數小時可以先練習的基礎演算法；就是先嘗試消除一組方塊(由中央開始，後續隨機進行)，然後遞迴進入下一層；如果發現無子可動（死局），就退回上一步嘗試另一組對子。

2. A* Search (暫定為100格地圖的推導方法)
我需要設計一個啟發式函數；啟發式函數先統計最一開始的地圖中有多少組可解的方塊，接著再嘗試推估第一步驟消除某組方塊的未來效益。效益的減項是不可解的方格配對數量；若是一開始的操作不能讓可以消除的配對增加，那麼就是屬於不太有效益的方式。

目前暫定針對第二部分的路徑搜尋以這兩種演算法進行，但是接下來一個禮拜我也會找尋更多可能性，嘗試有沒有更好的演算法來更快(似乎也只能在時間複雜度上變得更好)找出一個完全解。
