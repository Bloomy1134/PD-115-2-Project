# 消消樂遊戲的可解性與最佳解

# 動機與目標

消消樂是一款近期興起的益智遊戲；較少為人知，但是解題難度也較魔術方塊高。遊戲格式如下:

在這個遊戲中，玩家擁有兩種消除同顏色方塊的方式 : 直角消除與行列 "間隔" 消除。消除方式如下圖標示 :

遊戲中只要玩家點擊兩同色方塊在表格的直角交界處(此方式唯一)，或是在彼此同行列相間的任意格子(間隔的格子全部都可以點擊，消除效果相同)，便可以把方塊消除。空出來的格子也可以供消除其他方格來使用。

遊戲目標是將所有格子消除，全部方格共200格。若撇除個人圖形思考能力優劣所產生的影響，能破關遊戲的方式便在於；如何系統性的消除各個方塊，以使得最終不會淪為 "無子可動" 的窘境，如下圖 :

由上兩圖結算畫面可知，我無法找到任何同色方塊的直角交界點(路徑中有其他方塊)；或者同色方塊在同行列有間隔的格子。

同時，遊戲是有通關分數條件 (即200分) 。因此，遊戲本身應該有自有的地圖種子，來確保每張圖即便對於個體來說隨機；但都可以有解。

目前我遊玩次數約達100次，僅有一次成功全數消除獲得200分。且有5~6次消除到最後成為 "無子可動"。

本報告的研究動機在於；我對於一個消消樂地圖有完美解 (或無解) 的條件，以及是否有系統性解法感到好奇。魔術方塊玩家都有解題公式可以背誦；而這個遊戲也不能只依靠對於圖形之間相間關係的觀察敏銳度；而是要刻意去製造空間給下一次操作，確保一種顏色方格永遠有另一個同色方格來供兩者同時消除。此遊戲的方格數量遠超於一般魔術方塊且操作方式更加自由(畢竟一張圖可以有超過一組解)，因此推導的過程會更加獨特且複雜。

因此，我在這份專案中(理想情況下)想要製作兩種演算法。(以下接續其功能)

# 預期功能

1. **可以不需要紀錄解題路徑便可以判別地圖是否有完美解的演算法** : 當我輸入一個隨機生成的地圖時 (包含遊戲內目前所知的所有元素，但是每種顏色格子的擺放方式與數量皆不盡相同，測試資料都是由亂數決定的結果)，演算法可以在不直接確認執行步驟下，推論出這張地圖的所有方格是否可以只透過遊戲內指定的兩種方式去全數消除 (也就是說，我只讀入顏色方格內容，但是電腦不需要實際操作到最後全部消除；只要純看地圖內容就知道是否有解)。此演算法難在理論證明，因為我需要能依靠個人能力證明一張地圖有解的特質 (與為甚麼)，這樣就只需將這些特質作為條件確認，輸出 "有解" 或 "無解" (顯而易見的測試方式，就是至少當我直接依照正式遊戲內的既定地圖去輸入地圖情形；因為外面網站中的遊戲地圖保證有解，那因此這個演算法理當輸出 "有解")。

2. **尋找一個可行路徑的演算法** : 若我無法在時間內證明一張地圖是否可以在不實際執行操作的情況下確認有無完美解 (或是根本不存在我理想中的萬用確認方式)，那第二種演算法就是去實際跑很多遍解題流程，只依靠前述兩種方式去自行消除方塊，直到最後確認結果。由於這個遊戲的方格很多，可能運算的時間花費極大才能找到一個解。因此這時候我目前預期中的處理方式有兩種: 1. **減少格子總數** (對目標達成來講比較好) 2. **限制運算時間** (可能會在某些情況無法達成目標，因為可能有解但是演算法能力不足無法在限制時間內找到，就跟人腦一樣)。第二種演算法必定會實作；原因是若我推導出第一種演算法的有解證明條件，那我也需要透過路徑搜尋的演算法去進行確認，證明兩種演算法在理論上與實務上都可以導出有解的結果。
